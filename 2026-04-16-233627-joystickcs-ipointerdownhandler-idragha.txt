  주요 변경 사항

  UniTask 루프 (MoveLoopAsync)
  - OnPointerDown에서 새 CancellationTokenSource 생성 후 루프 시작
  - UniTask.Yield(PlayerLoopTiming.Update, token) 으로 매 프레임 player.position

  토큰 관리 흐름
  OnPointerDown  →  _moveCts = new CancellationTokenSource()
                     MoveLoopAsync(_moveCts.Token).Forget()

  OnPointerUp    →  _moveCts.Cancel()   // 루프 탈출
                     _moveCts.Dispose()  // 리소스 해제
                     _moveCts = null

  OnDrag에서 이동 제거
  - 기존 코드는 OnDrag 안에서 player.position을 직접 갱신했는데, 이 방식은
  드래그 이벤트 빈도에 의존하기 때문에 제거하고 UniTask 루프가 일관된 프레임

  핵심 설계

  _MainTex_ST 사용 이유

  MaterialPropertyBlock에는 SetTextureOffset이 없습니다. Unity 내부적으로
  Texture의 Tiling/Offset은 _MainTex_ST라는 Vector4 하나로 관리됩니다.

  _MainTex_ST = Vector4(tilingX, tilingY, offsetX, offsetY)
                         ─────────────────  ───────────────
                             xy = Tiling         zw = Offset

  때문에 Tiling을 덮어쓰지 않으려면 Awake에서 원본 값을 저장해야 합니다.

  루프 흐름

  OnEnable  →  CancellationTokenSource 생성 + RailScrollLoopAsync 시작

  매 프레임 (Update 타이밍)
    _offset.y -= scrollSpeed * Time.deltaTime   // y축 스크롤
    GetPropertyBlock → SetVector(_MainTex_ST) → SetPropertyBlock

  OnDisable →  Cancel + Dispose  (오브젝트 비활성화 시 루프 종료)

  MaterialPropertyBlock 사용 이유

  material.SetTextureOffset은 인스턴스 머티리얼을 생성해 메모리를 낭비합니다.
  MaterialPropertyBlock은 원본 머티리얼을 공유하면서 렌더링 시점에만 값을
  오버라이드하므로 배칭이 유지되고 메모리도 절약됩니다.
